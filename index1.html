Sistema Operacional

É um software fundamental que gerencia os recursos de um computador, tablet ou smartphone, permitindo que o usuário interaja com o hardware (Hardware, em termos de computação, refere-se à parte física de um computador ou sistema eletrônico. É tudo aquilo que podemos tocar e que compõe o dispositivo, como o processador, a memória, o disco rígido, o teclado, o mouse, o monitor, etc. Em resumo, hardware são os componentes eletrônicos e físicos que tornam um computador operacional) e execute aplicativos. Ele atua como uma ponte entre o usuário e a máquina, facilitando a execução de tarefas e o acesso a funcionalidades do dispositivo.

  • Gerencia o hardware;
  • Permite a execução de programas;
  • Fornece uma interface;
  • Garante a segurança;
  • Gerencia arquivos;
  • Controla processos;
Sistemas operacionais de desktop: Windows, macOS, Linux. Sistemas operacionais móveis: Android, iOS.

Sistemas operacionais embarcados: Android TV, Tizen

Processo

Processos de TI são todas as etapas que compõe a construção e a entrega de um serviço da TI. Mapear e gerir os processos de TI de maneira eficiente ajuda a elevar a performance e o valor do serviço entregue ao cliente, bem como os processos da empresa como um todo.

Thread

Em computação, um thread, ou fio de execução, é a menor unidade de processamento que pode ser gerenciada pelo sistema operacional. Ele representa um fluxo de execução dentro de um processo, permitindo que um programa realize múltiplas tarefas simultaneamente. Threads compartilham recursos como memória com outros threads dentro do mesmo processo, o que facilita a comunicação e o compartilhamento de dados entre eles.

Em outras palavras:

Processo: É um programa em execução, com seus próprios recursos, como memória e arquivos.

Thread: É uma parte de um processo que pode ser executada de forma independente, como se fosse uma "tarefa" dentro do programa.

Exemplo: Imagine um editor de texto como um processo. Dentro desse editor, você pode ter threads para:

Mostrar a interface do usuário (digitar, formatar texto).

Salvar o arquivo em segundo plano.

Verificar a ortografia enquanto você digita.

Todos esses "subtarefas" podem ocorrer simultaneamente, sem que uma impeça a outra de funcionar, graças aos threads.

Software

Software é um termo abrangente para programas, dados e instruções que operam um computador ou dispositivo eletrônico. É a parte lógica, não física, que permite que o hardware (a parte física) execute tarefas específicas. Em essência, o software diz ao hardware como funcionar e o que fazer.

Mais detalhadamente:

Instruções e programas:

O software é composto por um conjunto de instruções, também conhecidas como programas, que são escritos em linguagem de programação e executados pelo computador.

Parte lógica:

É a parte "intelectual" do computador, responsável por processar dados e controlar o hardware.

Tipos de software:

Existem diversos tipos de software, como sistemas operacionais (Windows, macOS, Linux), aplicativos (editores de texto, navegadores, jogos), e softwares de desenvolvimento (ferramentas para criar outros softwares).

Relação com o hardware:

O software e o hardware trabalham em conjunto. O hardware executa as instruções fornecidas pelo software.

Exemplos:

Aplicativos como editores de texto, planilhas eletrônicas, navegadores de internet, jogos, e até mesmo sistemas operacionais como Windows e macOS são exemplos de software.

URI

URI significa Identificador Uniforme de Recurso (do inglês, Uniform Resource Identifier). É uma sequência de caracteres que identifica de forma única um recurso, seja ele um arquivo, uma página web, uma imagem, um vídeo, um serviço, etc., na internet. Pense nele como um endereço que aponta para algo específico na web. A URI é um conceito mais amplo que inclui tanto URLs (Localizadores Uniformes de Recursos) quanto URNs (Nomes Uniformes de Recursos).

URL (Uniform Resource Locator):

Especifica a localização exata do recurso, como um endereço de site (ex: https://www.exemplo.com/pagina1.html).

URN (Uniform Resource Name):

Identifica um recurso pelo nome, sem necessariamente especificar onde ele está localizado (ex: urn:isbn:0-395-36341-1, que identifica um livro pelo ISBN).

Em resumo, a URI é o termo geral para identificar recursos na web, enquanto URL e URN são tipos específicos de URIs que indicam a localização ou o nome do recurso, respectivamente. A diferença entre URI e URL, em uma tradução livre, é: Uma URI pode ser definida como um localizador, enquanto uma URL provê também informações sobre como este recurso pode ser acessado.

DNS

O DNS, ou Sistema de Nomes de Domínio, é um sistema que traduz nomes de domínio legíveis por humanos (como www.example.com) em endereços IP numéricos (como 192.0.2.44) que os computadores utilizam para se comunicarem na internet. É como uma lista telefônica da internet, onde você procura o nome do site e encontra o número de telefone (endereço IP) correspondente.

Como funciona o DNS:

  1. Solicitação: Quando você digita um endereço de site no navegador, seu computador envia uma solicitação ao servidor DNS.
  2. Pesquisa: O servidor DNS pesquisa em sua base de dados para encontrar o endereço IP associado ao nome de domínio.
  3. Resposta: O servidor DNS retorna o endereço IP para o seu computador.
  4. Conexão: Seu computador utiliza o endereço IP para se conectar ao servidor do site e carregar a página desejada.

HSTS ou HTTP

HSTS, ou HTTP Strict Transport Security, é um mecanismo de segurança que força o navegador a usar apenas conexões HTTPS (seguras) com um site, evitando ataques de downgrades e sequestro de cookies. Essencialmente, ele informa ao navegador que, em vez de permitir conexões HTTP inseguras, ele deve se conectar apenas através de HTTPS, mesmo que o usuário digite apenas o endereço HTTP na barra de endereço. Aqui está um resumo de como o HSTS funciona e por que é importante:

Como funciona:

Um cabeçalho HTTP chamado Strict-Transport-Security é enviado pelo servidor ao navegador junto com a resposta. Este cabeçalho informa ao navegador para sempre usar HTTPS ao se comunicar com o site, em vez de HTTP. O navegador então intercepta todas as tentativas de acessar o site via HTTP e as redireciona automaticamente para HTTPS. O navegador também armazena essa informação por um período de tempo (definido pelo cabeçalho max-age) para evitar a necessidade de checar o cabeçalho HSTS repetidamente.
